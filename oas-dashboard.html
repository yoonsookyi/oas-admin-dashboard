⚠ NQSConfig.INI 직접 편집 금지
NQSConfig.INI는 OAS Console이 관리하는 파생 파일입니다.
직접 편집한 내용은 Console에서 설정을 저장·활성화하는 순간 덮어써집니다.
반드시 아래 OAS Console을 통해 설정하세요.
① OAS Console에서 Usage Tracking 활성화
http://서버포트/ui 접속
→ 우측 상단 Navigator(≡) → Console
→ Advanced System Settings
→ Usage Tracking
→ Enable Usage Tracking 체크
활성화 후 아래 파라미터를 설정합니다:
파라미터 형식 예시
Connection Pool <DB명>.<Connection Pool명> UsageTracking.UTConnectionPool
Init Block Table <DB명>.<스키마>.<테이블> UsageTracking.UT_Schema.InitBlockInfo
Physical Query Table <DB명>.<스키마>.<테이블> UsageTracking.UT_Schema.PhysicalQueries
Logical Query Table <DB명>.<스키마>.<테이블> UsageTracking.UT_Schema.LogicalQueries
Max Rows 1 ~ 100,000 (0 = 무제한) 10000
※ OBIEE 12c에서 업그레이드한 경우 기존 설정이 유지되나 Console에 값이 표시되지 않을 수 있습니다.
② metrics_server.py DB 연결 환경변수 설정
start.sh 에서 아래 환경변수 export 후 metrics_server.py 재기동:
OAS_UT_USER=<스키마계정>   OAS_UT_PASS=<password>   OAS_UT_DSN=localhost:1521/<service_name>
